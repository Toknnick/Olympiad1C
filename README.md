# Международная олимпиада по программированию на платформе «1С:Предприятие 8». Первый тур 2026 года, заочная форма
## Описание проблемы
Инвестор владеет определённой криптовалютой и хочет
конвертировать её в другую, более перспективную. Прямая
конвертация между этими активами может отсутствовать. Однако
существуют децентрализованные биржи (DEX), позволяющие
обменивать одни криптовалюты на другие через цепочки
промежуточных обменов.
Необходимо в системе «1С:Предприятие» разработать программу,
которая позволит определить, возможно ли конвертировать
исходную криптовалюту в целевую через цепочки промежуточных
обменов и найти оптимальную цепочку. Основной критерий
оптимальности — максимизация итоговой суммы, получаемой в
целевой валюте. Если существует несколько цепочек, дающих одинаковую максимальную сумму,
то среди них следует выбрать цепочку с минимальным количеством обменов. Если и таких
несколько — можно выбрать любую из них.
## Исходные данные
### Заголовочная часть:
Список допустимых обменов криптовалют хранится в документе «Криптообменники» следующей
структуры.

* Номер документа (строка);
* Дата документа (дата);
* Источник получения информации (строка, например, URL или название сервиса).

Табличная часть:
* Номер строки (число);
* «Платформа/Протокол» (строка) — название DEX или протокола, где возможен обмен;
* «Криптовалюта исходная» (строка) — тикер исходной криптовалюты;
* «Криптовалюта целевая» (строка) — тикер целевой криптовалюты.
* Курс обмена (число) – указывает, какое количество целевой валюты можно получить за 1
единицу исходной (при обмене на данной платформе).
* Комиссия (число) – указывает, процент от суммы целевой валюты, удерживаемый
платформой за конвертацию.

В табличной части может содержаться примерно до 1000 строк
## Выходная информация
Необходимо сформировать отчет с названием «Отчёт о результате конвертации криптовалюты
через DEX».
В форме отчета предусмотреть следующие элементы:
* поле для выбора документа «Криптообменники»;
* текстовое поле для ввода тикера исходной валюты;
* числовое поле для ввода суммы в исходной валюте;
* текстовое поле для ввода тикера целевой валюты.
  
Отчёт генерируется автоматически после ввода исходной и целевой криптовалют.
